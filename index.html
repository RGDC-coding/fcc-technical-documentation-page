Introduction

Welcome to the Flexbox Properties Guide

This website is a simple, straightforward guide dedicated to helping you understand the essential properties of Flexbox in CSS. Flexbox, or the "Flexible Box Layout," is a powerful layout tool that allows you to create dynamic and responsive designs with ease. Whether you're new to Flexbox or need a quick reference, this guide covers all the key properties you'll need to master.

What You'll Learn

In this guide, you'll explore the core Flexbox properties that control the behavior and alignment of elements within a flex container. From setting up a flex container to managing how items align, wrap, and distribute space, this resource is designed to provide clear, practical examples to help you implement Flexbox in your projects with confidence.


Dive into the sections below to start learning how to make your layouts more flexible and efficient using Flexbox!

Properties for the Parent (flex container)
display

The display property in Flexbox is used to designate an element as a flex container, which then enables the flexbox layout model for its direct children.


Once an element is set to display: flex; or display: inline-flex;, all of its direct children become flex items, which can then be arranged and aligned using other Flexbox properties. 

          
            .container {
  display: flex; /* or inline-flex */
}

Note:

CSS columns have no effect on a flex container.

flex-direction

The flex-direction property in Flexbox determines the direction in which flex items are placed inside a flex container. It establishes the main axis along which the flex items will be laid out and controls whether they are arranged in rows or columns, as well as the order in which they appear.

.container {
  flex-direction: row | row-reverse | column | column-reverse;
}

row: The default value. Flex items are laid out horizontally from left to right (in a left-to-right writing mode).


row-reverse: Flex items are laid out horizontally from right to left, reversing the normal order.


column: Flex items are laid out vertically from top to bottom.


column-reverse: Flex items are laid out vertically from bottom to top, reversing the normal order.

flex-wrap

The flex-wrap property in Flexbox specifies whether flex items should wrap onto multiple lines when they exceed the size of their flex container. By default, flex items will try to fit onto a single line, but you can use flex-wrap to allow them to wrap if necessary.

.container {
  flex-wrap: nowrap | wrap | wrap-reverse;
}

nowrap: This is the default value. Flex items are laid out in a single line, and they will overflow the container if they don't fit.


wrap: flex items will wrap onto multiple lines, from top to bottom.


wrap-reverse: flex items will wrap onto multiple lines from bottom to top.

flex-flow

The flex-flow property in Flexbox is a shorthand that combines the flex-direction and flex-wrap properties into a single declaration. This allows you to set both the main axis direction and the wrapping behavior of flex items in one line of code.

.container {
  flex-flow: column wrap;
}

The justify-content property in Flexbox is used to align and distribute flex items along the main axis of the flex container. This property is particularly useful for controlling the spacing and positioning of items when there is extra space within the container.

.container {
  justify-content: flex-start | flex-end | center | space-between | space-around | space-evenly | start | end | left | right ... + safe | unsafe;
}

flex-start: Aligns flex items to the start of the main axis. This is the default value.


flex-end: Aligns flex items to the end of the main axis.


center: Centers flex items along the main axis.


space-between: Distributes flex items so that there is equal space between them. The first item is aligned to the start of the container, and the last item is aligned to the end.


space-around: Distributes flex items with equal space around them. This means that the space between items is the same, and there is also space between the items and the container edges. Note that the space at the edges of the container will be half the size of the space between items.


space-evenly: Distributes flex items with equal space between them, including at the edges of the container. All gaps between items, as well as the gaps between items and the container edges, are equal.


start: Aligns flex items to the start of the writing mode direction (replaces flex-start in certain contexts).


end: Aligns flex items to the end of the writing mode direction (replaces flex-end in certain contexts).


left: Aligns flex items to the left edge of the container, unless this alignment does not fit with the flex-direction.


right: Aligns flex items to the right edge of the container, unless this alignment does not fit with the flex-direction.


Note:

Keep in mind that browser support for some justify-content values can be inconsistent. For instance, older versions of Edge do not support space-between, and some values like start, end, left, and right are not yet supported in Chrome. For the most reliable results, use flex-start, flex-end, or center.


Additionally, you can use the safe and unsafe keywords with these values. The safe keyword ensures that elements won't be pushed off-screen in a way that prevents content from being scrolled into view, avoiding "data loss."

The align-items property in Flexbox aligns flex items within a flex container along the cross axis (perpendicular to the main axis). This property affects how individual flex items are positioned relative to each other and the container's cross axis

.container {
  align-items: stretch | flex-start | flex-end | center | baseline | first baseline | last baseline | start | end | self-start | self-end + ... safe | unsafe;
}

flex-start: Aligns flex items to the start of the cross axis (top of the container in a horizontal flex container).


flex-end: Aligns flex items to the end of the cross axis (bottom of the container in a horizontal flex container).


center: Centers flex items along the cross axis.


baseline: Aligns flex items along their baseline, which is the line on which text sits.


stretch: Stretches flex items to fill the container along the cross axis. This is the default value, making items expand to match the height of the flex container.


Note:

The `safe` and `unsafe` modifier keywords can be used with all the values for `align-items` to help ensure that content remains accessible and does not get pushed out of view. For details on browser support, visit MDN Web Docs.

The align-content property in Flexbox aligns flex lines within a flex container along the cross axis (perpendicular to the main axis). It is used when the flex container has multiple lines of flex items, which occurs when flex-wrap is set to wrap or wrap-reverse.

.container {
  align-content: flex-start | flex-end | center | space-between | space-around | space-evenly | stretch | start | end | baseline | first baseline | last baseline + ... safe | unsafe;
}

flex-start: Aligns the flex lines to the start of the cross axis (top of the container in a horizontal flex container).


flex-end: Aligns the flex lines to the end of the cross axis (bottom of the container in a horizontal flex container).


center: Centers the flex lines along the cross axis.


space-between: Distributes the flex lines evenly along the cross axis, with equal space between them and none at the start or end.


space-around: Distributes the flex lines with equal space around them. This means the space on the edges will be half the size of the space between lines.


space-evenly: Distributes the flex lines with equal space between them, including the space at the edges of the container.


stretch: Stretches the flex lines to fill the available space along the cross axis. This is the default value.


Notes:

Support for align-content values can vary across browsers. It’s generally well-supported in modern browsers, but always check compatibility charts if you need to ensure consistent behavior.


The align-content property is useful for controlling the distribution and alignment of flex lines in a container, enhancing layout flexibility especially in responsive designs.

gap, row-gap, column-gap

gap: 50px

gap: 100px

gap: 50px 100px

The gap property specifically adjusts the spacing between flex items. It affects only the space between items, not the spacing at the outer edges of the container.

.container {
  display: flex;
  ...
  gap: 10px;
  gap: 10px 20px; /* row-gap column gap */
  row-gap: 10px;
  column-gap: 20px;
}

You can think of the gap property as setting a minimum space between items. If there is extra space due to properties like justify-content: space-between;, the gap will only apply if the available space becomes smaller than the specified gap.


Additionally, gap is not limited to Flexbox; it also works with CSS Grid and multi-column layouts.

Properties for the Children (flex items)
order

The order property in Flexbox is used to control the sequence in which flex items appear within a flex container. By default, flex items are displayed in the order they appear in the HTML source code. However, the order property allows you to change this order without altering the HTML structure.

.item {
  order: 5; /* default is 0 */
}

Items with the same order revert to source order.

flex-grow

The flex-grow property in Flexbox is used to define how flex items should grow relative to each other when there is extra space available in the flex container. It controls the ability of flex items to expand to fill the available space within their flex container.


If all items have a flex-grow value of 1, any extra space in the container will be shared equally among them. However, if one item has a flex-grow value of 2, that item will try to take up twice as much of the available space compared to the others

.item {
  flex-grow: 4; /* default 0 */
}

Note:

Negative numbers are invalid.

flex-shrink

The flex-shrink property in Flexbox determines how flex items should shrink relative to each other when the flex container is too small to fit all items at their initial sizes. It controls the ability of flex items to reduce their size to prevent overflow.

.item {
  flex-shrink: 3; /* default 1 */
}

Note:

Negative numbers are invalid.

flex-basis

This property sets the initial size of an element before any extra space is allocated. It can be specified as a fixed length (e.g., 20%, 5rem) or using keywords. The auto keyword tells the browser to use the element’s width or height properties for sizing, a behavior that was previously managed by the deprecated main-size keyword.


The content keyword would size the element based on its content, but support for this keyword is limited, making it difficult to test and understand how related values like max-content, min-content, and fit-content behave.

.item {
  flex-basis:  | auto; /* default auto */
}

If flex-basis is set to 0, the extra space around content isn’t factored in. If set to auto, the extra space is distributed based on its flex-grow value. See this graphic.

flex

This property is a shorthand for setting flex-grow, flex-shrink, and flex-basis all at once. The second and third values (flex-shrink and flex-basis) are optional. The default setting is 0 1 auto. If you use a single value, such as flex: 5;, it automatically sets flex-basis to 0%, which is equivalent to setting flex-grow: 5;, flex-shrink: 1;, and flex-basis: 0%;.

.item {
  flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]
}

It is advisable to use this shorthand property instead of setting the individual values separately. The shorthand automatically handles the other settings in a smart way.

align-self

The align-self property in Flexbox allows you to override the alignment of a specific flex item along the cross axis, independently of the other flex items in the container. This property is used to control how an individual flex item is aligned within its flex container, providing greater flexibility in layout design.


Note:

Float, clear and vertical-align have no effect on a flex item.
